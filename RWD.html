Why we have Responsive Web Design in 250 words or less

Necessity is the mother of invention.

Once upon a time in a land far, far away, a person likely accessed the internet using a personal computer and viewed the search results on a standard-size monitor (usually just one monitor, I might add). A standard-size monitor meant a standard-size webpage layout, which made web design relatively straightforward for a web developer.

The land of standard-size monitors and layouts is long gone; our new land is filled with tablets, smartphones, and laptops, oh my! We live in a world where the number of ways a person accesses and views the information on the World Wide Web is growing exponentially every year. Today, it is quite common for a person to access a webpage from a personal computer and one minute later access the same webpage from his/her smartphone!

To mitigate this change in the variety of devices and web access and to prevent the near impossible feat of designing webpages for every possible size monitor and behavior, web developers created responsive web design. Yeah!

Simply put: Using a responsive web design takes the guesswork out of designing webpage layouts for multiple devices. Web developers donâ€™t have to predict how a user will access and view information on the web page. Said another way, responsive web design considers how a person will access information from a variety of platforms (e.g., mobile device or desktop computer) and behave while using his/her device (e.g., viewing screen in portrait or tablet mode, minimizing or maximizing the screen size/view).
